🎁 デジタル返礼品配布プラットフォーム 要件定義書（v2アップデート版）
1. プロダクト概要
コンセプト: クリエイター・ライバー特化型のセキュアなデジタルコンテンツ配布ツール（進化版ギガファイル便）。

提供価値: 流出を防ぐ強固なセキュリティ、受け取り側の「特別感」の演出、そして配布側（ライバー）の作業コストの劇的な削減。

ビジネスモデル（マネタイズ）: ライバーをターゲットとしたB2B2CのSaaSモデル。フリーミアム（例: 月50ファイルまで無料、無制限化・高画質化は月額サブスクリプション）を想定し、初期の普及ハードルを下げる。

2. 開発アプローチ
UI-Driven Development（UI先行開発）: 生成AI（v0等）を活用し、Bento Gridの管理画面やスマホの開封アニメーションなど、フロントエンドのモックアップから最速で構築する。

「触り心地」と「ギフト体験」を検証した後に、バックエンド（Supabase等）の認証ロジックを繋ぎ込むことで、手戻りを防ぎモチベーションを維持したままMVPを完成させる。

3. UI / UX とデザインシステム
本格的なSaaSクオリティを目指し、直感的で美しく、デバイスに最適化された体験を提供する。

世界観とカラー:

ベース: 白を基調とした清潔感のある「グラスモーフィズム（すりガラス）」×「Bento Grid（弁当箱レイアウト）」。

アクセント: エメラルドグリーン（ボタン等のCTAに適用し、視線を誘導）。

テーマ: ダークモードとライトモードの切り替えを初期実装。ダークモード時もネオン調のエメラルドグリーンが映える設計。

操作性（マイクロインタラクション）:

ホバー時の浮き上がり（Elevation）や光の反射エフェクト。

初回利用時のわかりやすいオンボーディング（点線枠でのドロップ誘導など）。

キーボード操作（アクセシビリティ）への対応。

デバイス別画面設計:

配布側（PC前提・管理画面）:

Bento Gridの広い画面を活かした設計。

事務的な「CSVインポート」機能は排除し、ドラッグ＆ドロップ（D&D）操作の気持ちよさに極振りする。

画面左側の「返礼品（ファイル）」を、右側の「宛先カード」にドラッグして落とすだけで、裏側で専用リンクが即座に生成される直感的な紐付け。

受け取り側（スマホ前提・閲覧画面）:

迷わせないシングルタスクの動線。

リンクを開くと「プレゼントボックスが開く」リッチなアニメーション（開封の儀）。

4. コア機能要件
キャンペーン（グループ）ベースの配布管理:

アプリ内で「〇〇ガチャ」などの箱（キャンペーン）を作り、そこに「ファイルプール」と「宛先プール」を登録して管理する。

パスキー（WebAuthn）による強固な認証と紐づけフロー:

Claim Link（クレームリンク）＋ ゲスト登録: リスナーの事前アカウント登録（メアド等）は不要。ライバーから送られたURLを初回に開いた際、端末の生体認証（指紋・Face ID等）を要求し、そのパスキーをURLと強固に紐づける。以降は本人以外絶対に開けない。

機種変更時の対応: システム的なアカウントリカバリーは実装せず、「見られなくなったらライバーにDMして新しいリンクをもらう」というコミュニケーションでの解決に割り切り、開発・保守コストを大幅に削減する。

安全で軽量なプレビュー機能:

音声: ブラウザ標準の <audio> タグを利用した、エメラルドグリーンのモダンな再生プレイヤー（重い波形生成は行わずサクサク動く設計）。

画像: クライアントサイド（ブラウザ上）での自動透かし（ウォーターマーク）合成。サーバーの画像処理コストをゼロに抑えつつ、サーバーへの元データ保存を回避してセキュリティを最大化する。右クリック・長押し保存も防止。

自動ライフサイクル管理と「期限のエンタメ化」:

アップロードから一定期間経過したファイルをクラウド側で自動削除し、運用コストを最適化。

これを単なるシステム都合にせず、「受け取り（閲覧）期限」としてリスナー側に可視化し、開封を促すFOMO（見逃しへの焦り）のエンタメ演出としてポジティブに転換する。

5. 拡張・API要件（将来構想）
APIファースト設計: 内部処理をAPI化し、将来的に外部のガチャサービスや配信プラットフォームと直接連携できる基盤を作る。

外部IDの保持: 対象者のデータとして、連携先のアカウントIDや決済IDをDBに保持できる構造にしておく。

6. 推奨技術スタック
フロントエンド・バックエンド: Next.js (App Router) + TypeScript

データベース・認証: Supabase（パスキー認証の基盤として活用）

ファイルストレージ: Cloudflare R2（エグレス通信料無料でインフラコストを最小化）

UI・アニメーション: Tailwind CSS + shadcn/ui + Framer Motion / dnd-kit（ドラッグ＆ドロップ実装）

画像処理: Canvas API / WebAssembly（クライアントサイドでのウォーターマーク処理用）
